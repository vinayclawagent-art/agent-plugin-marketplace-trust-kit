# Trial Operator Handoff Checklist: Agent Plugin Marketplace Trust Kit

Source: [[Grok Build Plugin Marketplace as Agent App Store]]  
Package: [[Agent Plugin Marketplace Trust Kit]]

## Purpose
A one-page operator handoff that tells the next real trial runner exactly which existing artifacts to fill, in what order, without implying that validation has already happened.

## Trial sequence
1. Open [[Agent Plugin Marketplace Trust Kit/Plugin Trust Review Packet]] and define the real target, constraints, and success criteria.
2. During the run, paste every proof link into [[Agent Plugin Marketplace Trust Kit/Plugin Trust Evidence Index]] before editing any public claim.
3. After the run, fill [[Agent Plugin Marketplace Trust Kit/Plugin Trust Promotion Decision Card]] from the proof, not memory.
4. Complete [[Agent Plugin Marketplace Trust Kit/Post-Trial Plugin Trust Debrief Template]] and list which README, prototype, infographic, or skill-draft claims should change.
5. Only then patch the package README or skill draft; leave template-ready wording in place if evidence is incomplete.

## Minimum evidence checklist
- [ ] Plugin identity, repo, publisher, version, and install surface captured.
- [ ] Permission/API scopes and local file/network access listed.
- [ ] Sandbox command, transcript, and failure/rollback notes linked.
- [ ] Install / sandbox / iterate / reject decision filled from evidence.
